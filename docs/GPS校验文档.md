## 4. 基础链路校验

这部分在自动报站恢复前也必须通过。

### 4.1 配置核对

必须确认：

1. `debugReplay.gpsSerialKey = gps`
2. `serial.gps.portName = ttyS5`
3. `serial.gps.baudRate = 115200`
4. `serial.gps.mode = real`

如果切过配置，要删除运行期缓存：

`/storage/emulated/0/Android/data/com.lhxy.istationdevice.android11/files/config/terminal-config.runtime.json`

### 4.2 日志核对

必须能看到：

1. `stty applied /dev/ttyS5 @115200`
2. `real open /dev/ttyS5 @115200`
3. `已绑定 GPS 串口监听: gps/ttyS5`
4. `gps raw sample` 中持续出现 `$GNRMC`、`$GNGGA`、`$GNGSA`

### 4.3 定位状态核对

通过标准：

1. `valid=true`
2. `fixQuality>=1`
3. `fixType=2(2D)` 或 `3(3D)`
4. `usedSatellites>0`
5. `latitude / longitude` 非空

未通过但不一定是代码问题的情况：

1. 有 NMEA，但一直 `valid=false`
2. `GSA` 一直是 `A,1`
3. `GGA` 的 `fixQuality=0`

这通常优先看天线、环境、冷启动等待时间。

### 4.4 页面核对

通过标准：

1. 首页 `GPS` 状态能切到“有效”
2. 首页卫星数和实时定位一致
3. 站点学习页“实时 GPS”经纬度同步刷新
4. 页面速度显示为 `km/h`
5. 海拔字段不出现 `M` 这种单位串误入值

## 5. 串口上报校验

这部分针对当前已恢复的 DVR 串口 GPS 上报。

### 5.1 前提

只有在下面条件同时满足时才校验：

1. `dispatchOwner = serial_rs232_1`

3. 周期上报日志持续出现 `DVR_GPS_REPORT`
4. 帧里经纬度不是空值，也不是 `0`

## 6. 旧项目一致性校验

这部分才是“完整复刻”的标准，不是只有串口通了就算完成。

1. 自动寻找最近站点

3. 对开 / 环线 / 防反三种线路属性分支
4. 友情提醒点触发
5. 角度过滤
6. 站前里程阈值过滤
7. GPS 定时上报
8. GPS 校时
9. 站点学习保存回数据库

### 6.2 完整复刻通过标准

只有下面都成立，才能算 GPS 功能完整复刻：

1. 选用 `L1` 后，系统能根据实时定位自动找到最近站点
2. 能自动判定进站 / 出站
3. 能随着车辆移动自动切到下一站
4. 友情提醒点能按坐标和里程触发
5. DVR 周期 GPS 上报持续正常
6. 页面展示、串口上报、自动报站三者使用的是同一份实时 GPS 数据

## 7. 现场核对顺序

现场联调固定按这个顺序走：

1. 先核对 `ttyS5 / 115200 / real`
2. 再核对 NMEA 是否稳定输出
3. 再核对 `valid=true` 是否出现
4. 再核对首页和学习页是否刷新
5. 如果走 DVR，再核对 `DVR_GPS_REPORT`
6. 最后才核对自动报站是否按 `L1` 正常推进

不要一上来就用 `101` 判断自动报站是否可用，因为 `101` 当前资源里没有站点坐标。

## 9. 坐标转换契约

这次把旧项目链路重新核过以后，坐标契约已经确认：

1. GPS 模组原始值来自 NMEA：
   - 纬度 `ddmm.mmmm`
   - 经度 `dddmm.mmmm`
2. 线路资源 `L1S.csv / L1X.csv / *Remind.csv` 里的坐标也是同一种度分格式。
3. 旧项目不是直接拿这两个原始值算距离，而是先在 `MainPresenter` 里把线路站点和友情提醒点统一转换成十进制度。
4. 自动报站时，实时 GPS 也先转成十进制度，再和已转换的资源坐标做最近点匹配。

所以现在新壳里的正确校验方式应该是：

1. 串口原始日志继续看 NMEA 原文是否稳定。
2. 自动报站联调时，不要直接拿 `11405.8252 / 2242.7232` 这种原始资源值做距离判断。
3. 应该确认资源消费层已经先完成了 `ddmm/dddmm -> decimal degrees` 转换。

如果后面现场出现“GPS 明明有效，但自动报站始终找不到最近站点”，优先先核这条，而不是先怀疑模组。
