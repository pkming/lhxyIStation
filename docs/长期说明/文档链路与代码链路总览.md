# 文档链路与代码链路总览

这份文档只解决一个问题：

- 某份文档读完之后，代码应该先看哪里

如果 [README.md](../README.md) 解决的是“先看哪份文档”，这份文档解决的就是“下一步该进哪个目录、哪个类、哪层代码”。

---

## 1. 先记住一条总链路

新壳现在的阅读和落地顺序，统一按这条线理解：

1. 根目录文档先定目标和边界
2. 模块文档再收敛到某一条业务链
3. `app/` 里看页面入口和跳转承接
4. `modules/domain/` 里看业务模块、状态对象和用例编排
5. `modules/protocol/` 里看纯协议编解码
6. `modules/device-api/` 和 `modules/device-m90/` 里看硬件接入
7. `modules/runtime/` 和 `config/` 里看运行时装配和配置生效

一句话就是：

`文档定方向，app 找入口，domain 找业务，protocol 找报文，device 找硬件，runtime/config 找装配。`

## 6. 建议使用方式

1. 不确定先看哪份文档时，先回 [README.md](../README.md)
2. 已经知道文档，但不知道代码落点时，回这份总览
3. 已经找到模块代码，但不知道上下游时，先看 `TerminalModuleHub`，再回对应模块文档
4. 页面问题先从 `app/` 找入口，业务问题先从 `modules/domain/` 找模块，配置问题先从 `ShellConfig*` 找生效链
