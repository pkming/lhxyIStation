## 当前阶段

当前阶段不是“继续大面积补代码”，而是：

1. 代码侧可封板。
2. 以真机联调、点击回归和日志证据补录为主。
3. 只有主清单里仍明确标成 `进行中` 的代码缺口，才继续开代码修改。
4. 当前唯一明确未闭合的代码缺口，是首页 `JHY` 客流统计源未接入；其它剩余项主要是验证项。

## 1. 当前差异主要看哪五类

### 1.1 页面结构和入口

重点不是“有没有一个能跳进去的页面”，而是：

1. 页面结构是不是和 `M90` 一致。
2. 首页、登录、菜单、线路选择、调度、文件管理这些正式主链入口是不是一致。
3. 返回链、收口动作、页面间状态回填是不是一致。

这部分先看 [../清单/M90对照迁移清单.md](../清单/M90对照迁移清单.md)。

### 1.2 按钮语义和状态反馈

重点看：

1. 同一个按钮在 `M90` 上到底触发什么。
2. 页面提示、顶部状态栏、司机状态、线路摘要、视频区提示是不是按 `M90` 语义在刷新。
3. 现在是不是还残留“旧 4.4 语义”或“新壳临时语义”。

### 1.3 设置保存和运行期生效

重点看：

1. 设置项分组、默认值、保存后是否立即生效。
2. 保存后是否需要重启、回页、重拉状态。
3. 运行期到底由哪套共享状态或配置承接，避免页面临时状态和真实配置脱节。

### 1.4 协议和硬件承接

重点看：

1. 串口、GPIO、GPS、DVR 触摸、摄像头模式是不是按 `M90` 当前规则承接。
2. 没有硬件证据闭环的项，先记待确认，不直接猜。
3. 旧资料只拿来辅助确认字段和历史语义，不直接替代当前 `M90` 对照结果。

### 1.5 下载、升级和后台动作

重点看：

1. 后台下载、APK 安装、资源包处理、首页提示是不是按 `M90` 行为闭环。
2. 页面层 operation 回收动作哪些已经承接，哪些还没补齐。
3. 这部分判断以 [M90下载模块功能梳理.md](M90下载模块功能梳理.md) 和 `build/tmp` 下的反编译结果为准。

## 2. 当前已经明确的边界

后面继续推进时，不要再反复摇摆下面这些边界：

1. 不再按 `4.4` 重新定义当前完成标准。
2. 不为了兼容把逻辑重新塞回页面层。
3. 不为了“像旧项目”新增一套并行业务实现。
4. 不把手册截图当成高于 `build/tmp` 反编译结果的第一依据。
5. 确认不了的项直接进入“待验证 / 暂缓”，不要口头默认为已对齐。

### 2.1 当前已经明确收口的 M90 口径

下面这些点后面不要再回到旧 4.4 或截图猜测口径：

1. 首页左侧固定列当前按 `M90` 八键口径收口：`报站 / 重复 / 切换 / 服务键 / 停止 / 线路选择 / 司机键 / 菜单`。
2. 左侧第二键是“重复”，不再串接线路选择；第六键是“线路选择”，不是旧文档里曾经出现过的“报警开/关”。
3. 司机键当前按 `M90` 口径走三态：未签到显示 `未登陆`，仅刷卡未拿到真实司机身份时显示 `登陆`，拿到真实司机身份后显示 `司机：姓名`；点击行为和显示态共用同一判断。
4. 菜单页左上角第一格是 `DVR`，进入视频监控页；`menu_voice_call` 在 `M90` 上实际是“密码设置”，不是菜单级语音通话入口。
5. 登录和密码体系当前按 `M90` 口径收口：用户名隐藏不参与实质校验，登录接受“用户密码”或超级密码 `915742`；文件管理仅允许超级密码会话进入；密码设置页负责验证和修改用户密码。
