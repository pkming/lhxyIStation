# 真机联调必要项

这份文档只保留真机联调前必须确认的东西。

## 1. 当前配置入口

配置加载优先级固定如下：

1. `terminal-config.runtime.json`
2. `terminal-config.local.json`
3. `terminal-config.template.json`

当前真机包已经通过本地覆盖文件切到 `M90-REAL`：

- `config/terminal-config.local.json`

设备上如果还保留旧的 runtime 配置，会覆盖打包配置。

## 2. 真机前必须做的事

1. 删除或重置设备上的运行期配置文件：
   `/storage/emulated/0/Android/data/com.lhxy.istationdevice.android11/files/config/terminal-config.runtime.json`
2. 安装新包。
3. 首次启动后确认日志里出现 `M90-REAL`，而不是 `M90-STUB`。
4. 确认底层日志标签已经从 `M90StubGpio / M90StubSerial` 切到 `M90RealGpio / M90RealSerial`。

## 3. 当前已经切到 real 的能力

下面这些能力已经在本地覆盖配置里切到 `real`，可以直接上真机测：

- 串口：`ttyS0 / ttyS2 / ttyS3 / ttyS4 / ttyS5 / ttyS7 / ttyS9`
- Socket：`jt808 / al808`
- GPIO：音频、IO1~IO5、喊话器、耳机检测
- Camera：`av_out / reverse / middle_door / monitor`
- RFID：优先走 M90 JNI / I2C-3
- Keyboard：`ttyS0`
- SystemOps：已接入 real 实现，但命令开关默认仍关闭

## 4. 当前没有切到 real 的能力

下面两项当前仍保持 `stub`，不是漏改，而是为了避免空配置直接报死：

1. `can`
   原因：还没有确认 `can0 / can1` 的真实 `devicePath` 或命令桥接。
2. `location`
   原因：当前 GPS 主链仍走 `ttyS5` NMEA，不走 `LocationManager`。

## 5. 真机联调时要重点确认的东西

### GPIO

当前 real GPIO 直接读写下面这类节点：

- `/proc/rp_gpio/gpio1b1`
- `/proc/rp_gpio/gpio1b2`
- `/proc/rp_gpio/gpio0d6`
- `/proc/rp_gpio/gpio1d0`
- `/proc/rp_gpio/gpio1d1`

要确认两件事：

1. 节点存在。
2. 应用进程有读写权限。

### 串口

当前 real 串口实现直接打开 `/dev/ttySx`。

要确认三件事：

1. 节点存在。
2. 波特率和真实外设一致。
3. 系统镜像允许应用访问节点。

### Camera

当前 real Camera 直接走 Camera2 打开配置里的 `cameraId`。

要确认两件事：

1. `100/101/102/103` 这些 ID 在当前设备上真的存在。
2. 画面预览和切换不报权限或设备断开错误。

### RFID

当前 real RFID 优先依赖 `i2c-port` JNI。

要确认三件事：

1. 设备镜像里带 `i2c-port` 库。
2. `/dev/i2c-3` 可用。
3. 地址 `0x00` 是否真是当前设备值；如果不是，要改配置。

### SystemOps

当前 `systemOps.mode` 已切到 `real`，但以下能力默认还没打开：

- 静默安装
- 重启
- 校时

原因不是实现没接，而是还没有确认真机命令模板。

如果要测，需要补这些字段：

- `systemOps.silentInstallCommand`
- `systemOps.rebootCommand`
- `systemOps.setTimeCommand`

## 6. 日志验收口径

真机联调至少看这几类日志：

1. 启动时的配置来源和 `deviceProfile`
2. `M90RealGpio` 的 `real write / real read`
3. `M90RealSerial` 的 `real open / real send / real recv`
4. `M90RealCamera` 的打开、断开、失败日志
5. `M90RealRfid` 的 JNI 初始化和读卡日志

如果日志里还在大量出现 `M90Stub*`，优先怀疑配置没有真正切过去，而不是业务逻辑本身有问题。

## 7. 当前最短操作路径

1. 用当前仓库直接打包：`./gradlew assembleRelease`
2. 删设备上的 `terminal-config.runtime.json`
3. 安装新包并启动
4. 导出一份新日志
5. 先确认是不是全链路已经从 `stub` 切到了 `real`
