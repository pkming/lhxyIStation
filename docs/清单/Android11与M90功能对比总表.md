# Android11 与 M90 功能对比总表

这张表用于把 `Android11` 当前实现、`M90` 正式基线、历史 `old/4.4/K80` 做法放在同一张账里看。

当前判定口径：

1. `M90` 是当前唯一正式基线。
2. `old/4.4/K80` 只做历史参考，不作为完成标准。
3. Android11 允许保留新架构分层，但页面、动作、状态、协议语义要按 M90 收口。
4. “进度”是当前代码和文档口径的工程进度，不等同于真机 100% 复刻。

进度标记：

| 进度 | 含义 |
| --- | --- |
| `100%` | 页面、动作、状态、协议和证据均已闭合，当前无影响结论的待确认项 |
| `90%` | 代码逻辑已按 M90 收口，仍需真机观感、硬件或联调证据 |
| `80%` | 主链已接通，但仍有明确真机/协议/资源限制 |
| `60%` | 已有承接点，仍有明确代码或数据源缺口 |
| `0%-50%` | 非正式链路、历史保留项或尚未按 M90 定范围 |

> `90%` 的含义不是“源码逐行完全一样”。当前项目按 M90 复刻功能语义、协议字段和页面行为；native 串口、I2C、GPIO、DVR、系统设置等底层访问仍走 Android11 的统一适配层。凡需要真实口位、电平、cameraId、设备回包、平台联调或完整资源包才能证明的项，统一先停在 `90% / 待验证`，真机闭合后再升为 `100%`。

## 7. 后续维护规则

1. 后续每发现一个差异，先补到本表对应行，不要只留在会话里。
2. 如果是代码缺口，写清 Android11 承接点和下一步要改的类。
3. 如果是 M90 未做或证据不足，不新增 Android11 功能，只记录为暂缓/待确认。
4. 如果是 old 与 M90 冲突，默认 M90 胜出，old 只写在历史列里。
5. `100%` 只能在代码、状态、协议、真机证据都闭合后填写。
