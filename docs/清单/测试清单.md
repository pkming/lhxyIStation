# Android11 和 M90 功能校验表

这个表主要是给 Android11 这边做功能核对用的。

简单说，就是看 Android11 现在做出来的东西，和 M90 正式版本是不是一致。

这里不是要求 Android11 的代码和 M90 一模一样，而是看最终效果：

1. 页面是不是一样。
2. 按钮点了之后是不是一样。
3. 状态显示是不是一样。
4. 串口、GPS、刷卡、视频、GPIO、客流这些功能是不是按 M90 的逻辑跑。
5. 需要真机测试的地方，要补测试结果，不能只看代码。

---

## 1. 校验标准

| 项目 | 说明 |
| --- | --- |
| 正式参考版本 | 现在只按 `M90` 来对 |
| 旧版本参考 | `old / 4.4 / K80` 只看历史做法，不作为最终标准 |
| Android11 代码结构 | Android11 可以保留现在的新结构，不要求照抄 M90 旧代码 |
| 功能标准 | 页面、按钮、状态、数据、设备行为要和 M90 对齐 |
| 当前进度 | 表里的进度只是当前代码和文档判断，不代表真机已经完全通过 |
| 100% 标准 | 必须代码、页面、状态、协议、真机测试都没问题，才能写 100% |

---

## 2. 进度说明

| 进度 | 含义 |
| --- | --- |
| `100%` | 功能、页面、状态、设备测试都确认没问题 |
| `90%` | 代码基本做好了，但还需要真机确认 |
| `80%` | 主流程已经通了，但还有设备、资源或协议问题没确认 |
| `60%` | 已经有代码入口，但还缺明显功能 |
| `0%-50%` | 不是正式功能，或者还没按 M90 做完 |

说明：

`90%` 不代表完全完成。  
很多功能必须接真机才能确认，比如串口、GPS、DVR 视频、刷卡、GPIO、电平、客流设备回包等。  
这些功能代码接好了，也只能先写 `90% / 待验证`。  
等真机确认没问题后，才能改成 `100%`。

## 3. 校验结果怎么填

| 结果 | 含义 |
| --- | --- |
| `通过` | 和 M90 一致，测试证据也够了 |
| `待真机确认` | 代码已经接了，但还要用真机测 |
| `不通过` | 和 M90 明确不一致 |
| `阻塞` | 缺设备、缺资源、缺账号、缺权限、缺串口映射等，暂时没法测 |
| `不纳入` | 历史保留页面、调试页面，不算正式 M90 功能 |
