# AL-M90 联调资料索取清单

更新时间：2026-05-13

## M90 核对结论

M90 不是都没做。能确认已经做了的有：RS485 主链、通达/TD 屏显协议主链、RFID I2C 刷卡、喊话 GPIO 组合逻辑、耳机系统广播监听、摄像头切换、公交调度平台 Socket 主链、系统权限和签名。

RS485 的协议和波特率属于本地固定配置项，外设设置页选完就保存；后续发送按保存后的 RS485-1 / RS485-2 配置执行。TD/LED 屏显也跟随这里选择的协议走 M90 对应实现，不再作为资料索取项。喊话按 M90 的 `gpio3a4 / gpio3a3` 四种组合逻辑实现，也不再作为资料索取项。

SOS 当前也按 M90 现状处理。既然 M90 里没看到独立 SOS 主链，这一版先不单列索取、不额外外扩。

系统权限和签名这一版也直接按 M90 口径处理，不再单列索取。

摄像头业务通道这一版也按 M90 对照实现：DVR / monitor 走 `Camera.open(0)`，倒车走 `Camera.open(1)`，中门走 `Camera.open(2)`，DVR 通道切到 `VIN4-AUTO` 时改走 `Camera.open(3)`；因此不再单列索取。

M90 自己是有公交调度平台主链的：网络设置里能看到协议、服务器 IP、端口、心跳等参数，代码里也有 `NetDispatchProtocol / NetDispatchIP / NetDispatchPort` 对应的 Socket 连接和 808/ALINK 类协议打包上报链。

第三方平台接口来自资料包里的独立《公交平台-第三方平台对接协议文档》，协议形态是 WebService + JSON/Base64/GZip；这不是 M90 当前公交调度主链，而是另一套外部接口资料。当前 M90 / old / Android11 主链都没看到这套正式客户端，所以这一版也不按对齐目标单列索取。

RFID 这一项也按 M90 口径处理：既然现板确认可以直接复用 M90 的 `I2CPort + libi2c-port.so`，这一版就不再单列索取。

M90 没看到完整业务主链的有：IO3 / IO4 / IO5、ttyS0 按键协议、CAN 业务收发、SOS 完整链路。

M90 做了一半但仍要问清楚的有：耳机 GPIO 在插拔前后的实际电平变化。
