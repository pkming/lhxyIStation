## 2026-04-20 - 文件管理补 SourceFile.zip 与内置资源导入兜底

### 背景

- 当前文件管理页导入报站资源时，主格式就是 `SourceFile.zip`。
- 这次仓库里新增了正式资源文件 `docs/SourceFile.zip`，需要让它既能作为外部导入源，也能作为安装包内置兜底资源。
- 同时保留旧 `SourceFile.rar` 的扫描提示，但不恢复自动解包。

### 本次落地

- `StationResourceArchiveUseCase` 继续以 `SourceFile.zip` 作为正式导入格式。
- `app/build.gradle` 新增内置资源准备任务：如果仓库里存在 `docs/SourceFile.zip`，构建时会一起打进 app assets。
- 外部导入目录没有资源时，文件管理导入会继续尝试读取安装包内置的 `SourceFile.zip` 作为兜底来源。

### 当前结论

- 现在报站资源导入有两条可用路径：
  - 手动把 `SourceFile.zip` 放进导入目录。
  - 直接使用随安装包带入的 `docs/SourceFile.zip` 内置资源。
- 旧 `rar` 仍未恢复自动解包；如果后面拿到的还是 `rar`，建议先转成 `zip`。

## 2026-04-17 - 基础设置补资源导入统一状态源

### 背景

- 旧项目基础设置保存前依赖“报站资源已导入”前置判断。
- 新壳此前没有正式状态源，导致这条旧行为只能停留在文档说明里。

### 本次落地

- 新增 `LegacyStationResourceStateRepository`，把“报站资源已导入”收口成统一状态。
- `LegacyFileManageActivity` 的导入按钮现在会写入这个占位状态，并展示当前线路、来源和时间。
- `LegacyBasicSetupSectionFragment` 的报站 / 网络 / 串口 / TTS / 语言 / 其他保存入口已统一读取这个状态，不满足时直接提示“请先导入报站器资源”。

### 当前结论

- 基础设置旧前置现在已经不是口头约定，而是正式代码路径。
- 资源导入状态现在已进入运行期配置，重进页面和导出调试包时也能复用同一状态源。
- 文件管理页现在已支持真实扫描旧目录 BusRes/BusImport 和 app 专用 imports 目录；找到 SourceFile.zip 后会解压到运行目录并写回统一资源状态。
- 当前仍未恢复 rar 解包和旧数据库落库，后续真实旧业务链路接回时继续复用同一状态源。
- 网络/串口互斥也已增加 dispatchOwner 正式状态，用来表示当前由网络还是 RS232-1 承接调度；该状态现在已继续接到第一批真实业务消费。
- 首页和 DispatchBusinessModule 已开始消费 dispatchOwner：切到串口调度时，不再继续把 socket 当成当前主链路展示和回放。
- 当 dispatchOwner=serial_rs232_1 且 RS232-1Protocol=DVR 时，DispatchBusinessModule 的确认调度 / 发车动作，以及 StationBusinessModule 的 GPS / 站点推进，都会开始按旧 DVR 帧结构发送到 RS232-1。
- 同一模式下，SignInBusinessModule 的刷卡签到 / 手动签退也会按旧 DVR 考勤帧结构发送到 RS232-1；这样调度、报站、考勤三块已经共用一套串口主链。
- 调度模块现在也补了“确认公告”动作：收到或生成待确认公告后，可按旧 DVR lowerReply 结构发送下发回复帧。
- 旧版视频监控页的数字键、方向键、菜单/确认/删除/返回键也已按旧键码映射接回，点击时会通过 camera_dvr 模块发送 DVR 键位帧。
- 旧版视频监控页的预览区也已开始采集触摸事件，并按文档摘要里的 `55 AA + 05 + X/Y + 按下/松开 + 校验` 结构发送 DVR 触摸帧；当前先按 M90 的 `1280x800` 坐标系缩放，坐标数值按现有 DVR 协议风格走小端打包，周末需重点核对这两个假设。
- 协议日志解释器也已补上 DVR/GPS 上报、DVR/站点信息、DVR/调度回复、DVR/发车上报摘要，周末看日志时不再只剩原始 HEX。
- 协议日志解释器已继续补上 DVR/司机考勤、DVR/下发回复、DVR/键位事件、DVR/触摸事件摘要。
- 视频监控页 UI 日志也已补到能直接看到触摸 `phase`、原始 View 坐标、View 宽高和换算后的协议坐标；周末排查触摸时先看 UI 日志，再看协议日志，不要直接猜协议值。
- 当前仍未恢复的是故障/充电等完整串口调度动作；DVR 触摸协议已接第一版，待真机核对坐标系和字节序。
