# UI 与业务核对清单

这份文档后面专门用来做一件事：

- 按旧 `4.4` 项目，一项一项核对新壳还差什么

这份文档的使用原则：

- 正式 UI 以旧 `4.4` 项目为基线
- debug 页和 debug 入口可以保留，但只保留在 debug 环境
- 后面核对时，先看页面，再看业务动作，再看协议和联调

---

## 1. UI 页面核对清单

### 1.1 UI 对齐原则

#### 1.1.1 总原则

##### 1.1.1.1

正式 UI 必须和旧 `4.4` 终端一致，不能长期保留现在这种“新壳驾驶台 + 通用模块页”作为最终界面。

#### 1.1.2 debug 原则

##### 1.1.2.1

`BusinessModulesActivity` 和 `DebugToolsActivity` 可以继续保留，但只能作为 debug 路径，不算正式 UI。

#### 1.1.3 基线固化原则

##### 1.1.3.1

在真正开始 `1:1` 对齐前，先固化旧版基线，至少要保留：

1. 首页 / 登录 / 菜单 / 调度 / 报站 / 签到 / 视频 / 文件 / 设置 / 系统信息的截图
2. 关键主流程录屏
3. 按钮 -> 页面反馈 -> 弹窗 -> 语音 / 屏显 / 状态变化的对照
4. 关键文案、颜色、字体和按钮位置

### 1.4 首页 UI 差异

1. 报站
2. 线路
3. 切换
4. 键盘
5. 停止
6. 报警开
7. 菜单

##### 1.4.1.3 中间和右侧

旧首页中间和右侧是固定业务区：

1. 线路信息
2. 司机信息
3. 发车时间
4. 速度 / 限速
5. 数字键盘区
6. 其他业务显示区

#### 1.4.2 新壳首页现状

##### 1.4.2.1

新壳首页现在更像“业务总览页”：

1. 顶部是终端状态和配置摘要
2. 中间是调度 / 报站 / 签到 / 视频 / 系统 / 配置卡片
3. 下面是六个业务入口按钮
4. debug 入口在 debug 区域里

#### 1.4.3 首页核对结论

##### 1.4.3.1 已部分对齐

1. 已经不是开发摘要页
2. 已经能展示调度 / 报站 / 签到真实状态

##### 1.4.3.2 还没对齐

1. 顶部黑色状态栏没还原
2. 左侧固定按钮列没还原
3. 数字键盘区没还原
4. 主业务区布局没还原
