# 配置管理

## 1. 为什么单独放配置

这个项目后面会反复改这些东西：

- 串口口位
- 波特率
- 调度 host / port
- 调试页默认走哪条链路

如果还像旧项目那样把值散在代码里，后面会有几个问题：

- 不知道到底哪份值生效
- 改 host / port 还要改代码
- 运维、测试、开发没法对同一份配置说话
- 真机联调时容易把临时值带进正式代码

所以新壳这里统一改成：

- `config/` 专门放运行配置
- 构建时自动打包进 `assets`
- 代码只读配置仓库，不直接写死 host、port、串口号

## 2. 目录约定

```text
IStationDevice-Android11/
├── config/
│   ├── README.md
│   ├── terminal-config.template.json
│   └── terminal-config.local.json   # 可选，本地覆盖，不进仓库
```

## 3. 生效顺序

构建时按这个顺序取配置：

1. 运行期覆盖文件 `terminal-config.runtime.json`
2. 构建打包进去的 `terminal-config.local.json`
3. `terminal-config.template.json`
4. 如果文件异常，代码回退到默认内置配置

这样就算本地没配，也不会影响 APK 先跑起来。

应用第一次启动时，会自动把当前打包配置复制到运行期文件：

- `Android/data/<包名>/files/config/terminal-config.runtime.json`

后面联调优先改这份，再到调试页点“刷新”。
调试页和导出的调试包里也会直接带出这份文件的位置，方便现场确认到底改的是哪份配置。
如果现场要回退到打包时的标准配置，直接在调试页点“重置运行配置”即可，不用手工删文件。

## 4. 当前配置结构

当前统一分 8 块：

- `serial`
  - 串口名
  - 波特率
  - mode
  - 备注
- `socket`
  - 通道名
  - host
  - port
  - mode
  - 备注
- `gpio`
  - mode
  - 逻辑 pin key
  - 物理 pinId
  - valuePath
  - 默认值
- `camera`
  - mode
  - 逻辑通道 key
  - cameraId
- `rfid`
  - mode
  - mockCardNo
  - inputFilePath
  - readCommand
- `systemOps`
  - mode
  - 是否支持静默安装
  - 是否允许重启
  - 是否允许校时
  - 各类命令模板
- `basicSetup`
  - 报站设置
  - 网络页补充参数
  - 串口协议偏好
  - TTS 设置
  - 语言设置
  - 其他音量设置
  - 无线入口开关
- `debugReplay`
  - 屏显默认走哪条串口
  - GPS 默认绑哪条串口
  - JT808 默认走哪个 socket
  - AL808 默认走哪个 socket
  - 默认 GPIO 选哪项
  - 默认 Camera 选哪项

当前默认调度通道除了回放用途，也会在应用启动时挂上 Socket 协议监听。

系统设置页后面凡是“工作日离线就能保存和回显”的项，统一优先落在 `basicSetup`。
这样页面不需要各自保存一份状态，运行期配置和调试包也能直接带出来。

## 5. 使用原则

- 要改口位，先改配置文件。
- 要改 host / port，先改配置文件。
- 要切串口 stub / real，先改 `serial.mode`。
- 要切 stub / real，先改 `socket.mode`。
- 要改 GPS 默认监听口，先改 `debugReplay.gpsSerialKey`。
- 要切 GPIO stub / real，先改 `gpio.mode`。
- 要切 Camera stub / real，先改 `camera.mode`。
- 要切 RFID stub / real，先改 `rfid.mode`。
- 要切系统能力 stub / real，先改 `systemOps.mode`。
- 要补 GPIO 真路径，先填 `gpio.pins.*.valuePath`。
- 要补 Camera 映射，先改 `camera.channels.*.cameraId`。
- 要补 RFID 桥接，先改 `rfid.inputFilePath` 或 `rfid.readCommand`。
- 要补系统命令，先改 `systemOps.rebootCommand / setTimeCommand / silentInstallCommand`。
- 页面和用例层不直接写 `ttyS7`、`7000` 这种值。
- 真机临时联调用 `terminal-config.local.json`。
- 真机在线联调优先改 `terminal-config.runtime.json`。
- 模板文件只保留当前标准值。

## 6. 注释要求

配置相关代码统一按老项目那种完整注释写法：

- 类上写“这个类干什么”
- 公开方法写“做什么、参数是什么”
- 容易误解的默认值和回退逻辑要写清楚

后面新增：

- 配置模型
- 配置加载器
- 配置仓库
- 设备适配器接口

都按这个标准补注释，不留裸类。
