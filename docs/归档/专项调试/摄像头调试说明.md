# 摄像头调试说明

这份文档只解决一个问题：

- 现在摄像头链路已经接进来了，真机上应该怎么调

它不和 GPS、DVR 触摸、站点学习混在一起看，专门给摄像头 / 视频监控联调用。

---

## 1. 当前代码链路

当前和摄像头直接相关的入口有：

1. `LegacyMainActivity`
   - 首页 `DVR` 按钮进入视频监控页
   - 首页 `DVR` 状态位现在看的是 `DvrSerialMonitor` 握手在线状态
2. `MainActivity`
   - 新业务首页没有单独的 `DVR` 按钮
   - 这里走的是 `视频 / DVR` 入口，最后也会进入视频监控页
3. `LegacyVideoMonitorActivity`
   - 视频监控页
   - 进入页面时触发默认 Camera 打开
   - 退出页面时触发默认 Camera 关闭
   - 页面内还有 DVR 键位和触摸协议发送
4. `CameraDvrBusinessModule`
   - 承接默认 Camera 开关、DVR 键位、DVR 触摸事件
5. `M90ManagedCameraAdapter`
   - 根据配置切 `stub / real`
6. `M90RealCameraAdapter`
   - 真机 Camera 打开 / 关闭入口
7. `DvrSerialMonitor`
   - DVR 串口握手和入站帧监听

所以现在真机调摄像头，不是只看页面有没有进来，而是至少要同时看：

1. 配置
2. Camera open / close 日志
3. 通道号是否正确
4. 如果走 DVR，还要看首页 `DVR` 在线状态

---

## 2. 调试前先核对什么

先看运行期配置文件：

`/storage/emulated/0/Android/data/com.lhxy.istationdevice.android11/files/config/terminal-config.runtime.json`

至少确认下面几项：

1. `camera.mode = real`
2. `debugReplay.cameraChannelKey = 当前要调的通道`
3. `camera.channels.<key>.cameraId = 真机实际 cameraId`
4. 如果同时调 DVR 串口：`serial.rs232_1.mode = real`
5. 如果要看首页 `DVR` 状态：`basicSetup.serialSettings.rs2321Protocol = DVR`

一个最小可调的参考配置长这样：

```json
{
  "serial": {
    "rs232_1": {
      "portName": "ttyS3",
      "baudRate": 9600,
      "mode": "real",
      "note": "DVR / RS232-1"
    }
  },
  "camera": {
    "mode": "real",
    "channels": {
      "av_out": {
        "cameraId": "7",
        "note": "AV-OUT / DVR"
      },
      "reverse": {
        "cameraId": "4",
        "note": "倒车"
      },
      "middle_door": {
        "cameraId": "5",
        "note": "中门"
      },
      "monitor": {
        "cameraId": "6",
        "note": "监控"
      }
    }
  },
  "debugReplay": {
    "cameraChannelKey": "av_out"
  },
  "basicSetup": {
    "serialSettings": {
      "rs2321Protocol": "DVR"
    }
  }
}
```

上面这组 `cameraId` 不是凭空写的，是旧项目里的参考映射：

1. `av_out = 7`
2. `reverse = 4`
3. `middle_door = 5`
4. `monitor / DVR = 6`

先按这组试，最后以当前设备实测为准。

---

## 3. 建议的真机调试顺序

不要四路一起改，固定按这个顺序：

1. 先调 `av_out`
2. 再调 `reverse`
3. 再调 `middle_door`
4. 最后调 `monitor`

每一路都按同一套动作走：

1. 改好运行期配置
2. 重进应用
3. 从当前首页进入视频监控页
   - 如果是旧首页：点 `DVR`
   - 如果是新业务首页：点 `视频 / DVR`
4. 看默认 Camera 是否打开
5. 退出页面
6. 再进一次
7. 同一路连续进出 3 次

如果第一路都没稳定，不要往后调第二路。

## 4. 现场最该看的日志

### 4.1 Camera 日志

看 `M90RealCamera`：

1. `camera opened:`
2. `camera closed:`
3. `camera open failed`

这三类最关键。

### 4.2 页面层日志

看 `LegacyVideoMonitorActivity`：

1. 页面进入
2. 键位点击
3. 触摸坐标换算

### 4.3 模块层日志

看 `CameraDvrBusinessModule`：

1. 当前默认 Camera key
2. 当前 DVR 监听状态

### 4.4 DVR 串口日志

看 `DvrSerialMonitor`：

1. `已绑定 DVR 串口监听`
2. `DVR 在线`
3. 原始入站帧

如果 Camera 有画面但 DVR 状态一直不在线，不要混成一个问题看。

---

## 5. 用 ADB 怎么看

先确认设备在线：

```powershell
adb devices
```

实时看关键日志，Windows 下可以直接这样：

```powershell
adb logcat | findstr "M90RealCamera LegacyVideoMonitorActivity CameraDvrBusinessModule DvrSerialMonitor M90RealSerial"
```

如果只想看 Camera：

```powershell
adb logcat | findstr "M90RealCamera LegacyVideoMonitorActivity"
```

如果同时看 DVR 串口握手：

```powershell
adb logcat | findstr "DvrSerialMonitor M90RealSerial"
```

如果要重新推运行期配置，可以先准备好本地文件，再推到设备：

```powershell
adb push config\\terminal-config.local.json /sdcard/Android/data/com.lhxy.istationdevice.android11/files/config/terminal-config.runtime.json
```

推完后重进应用，再开始看日志。

## 6. 典型问题怎么判断

### 6.1 没有 `camera opened`

优先查：

1. `camera.mode` 是不是还在 `stub`
2. 应用有没有 Camera 权限
3. `cameraId` 对不对

### 6.2 有 `camera opened` 但没有画面

优先查：

1. 通道号映射
2. 真机接线
3. 预览链路

这时候先不要回头改页面布局。

### 6.3 只有某一路异常

优先怀疑：

1. 该路 `cameraId`
2. 该路接线
3. 该路硬件本身

不要先怀疑整条 Camera 链路。

### 6.4 画面有了但 DVR 不联动

再往下看：

1. `DvrSerialMonitor` 是否在线
2. 首页 `DVR` 状态是否同步变化
3. DVR 键位协议有没有发出
4. DVR 触摸协议有没有发出

---

## 7. 这一轮最小通过标准

摄像头这块，这一轮现场先按下面 5 条算通过：

1. 进入视频页时有明确 `camera opened`
2. 退出视频页时有明确 `camera closed`
3. 同一路连续进出 3 次都稳定
4. 改 `cameraChannelKey` 或 `cameraId` 后，现象会跟着变化
5. 如果走 DVR 模式，首页 `DVR` 状态能跟握手在线状态一致变化

---

## 8. 现在还不要混在一起看的东西

现阶段这些问题先不要和“摄像头有没有打开”混成一个问题：

1. DVR 触摸坐标字节序
2. 倒车 / 中门专用事件帧
3. GPIO 被动触发切画面
4. DVR 入站 CAN / 专用协议事件

先把 Camera 打开 / 关闭和通道号跑顺，再收这些链路。
