## 2. 当前明确不要继续扩的点

### 2.1 旧版没有的功能

统一先不做，不在这轮迁移里扩范围。

### 2.2 喊话链路

当前不要硬接，原因已经明确：

1. 旧版喊话依赖 `h19/h20` 组合输入。
2. 新壳当前还没有这两个输入在 M90 上的可靠映射。
3. [app/src/main/AndroidManifest.xml](../../../app/src/main/AndroidManifest.xml) 还没有录音权限。
4. [config/terminal-config.template.json](../../../config/terminal-config.template.json) 里也还没有对应的 GPIO key。

这类项统一看 [暂缓与待确认问题归档.md](../../暂缓与待确认问题归档.md)。

## 3. 下一步先做什么

### 3.1 推荐直接切到首页模块

当前最合适的下一步不是继续往硬件音频深挖，而是按现有路线继续收首页细项。

优先顺序：

1. 顶部状态栏真实状态
2. 左侧按钮细动作
3. 数字键盘剩余旧行为
4. 首页主业务区状态和反馈联动

原因：

1. 这块不依赖新增硬件映射，可以继续离线闭环。
2. 文档优先级当前也把首页细项放在第一组。
3. 音频里真正还没闭环的剩余项主要是喊话，而它已经被明确归档，不适合继续硬推。
