# 文档导航

这份文档只解决一个问题：

- 现在要做什么时，应该先看哪份文档

如果一时不知道从哪份开始，先看这份，再跳到对应文档。

---

## 1. 最常用的进入方式

### 1.1 新开会话，先接上当前进度

先看：

1. [新会话接手说明.md](新会话接手说明.md)
2. [推进步骤.md](推进步骤.md)
3. [项目架构.md](项目架构.md)

适用场景：

1. 刚开新会话
2. 隔了几天重新回来
3. 想先知道当前路线、当前做到哪、还差什么

### 1.2 要继续推进功能，不想再边做边换方向

先看：

1. [推进步骤.md](推进步骤.md)
2. [UI与业务核对清单.md](UI与业务核对清单.md)
3. [旧项目功能执行文档.md](旧项目功能执行文档.md)
4. [新壳与旧4.4差异记录.md](新壳与旧4.4差异记录.md)

适用场景：

1. 要决定下一步先补哪块
2. 要按旧 4.4 对齐页面和行为
3. 要判断这次是补 UI、补业务，还是补差异说明

### 1.3 工作日离线推进，没有真机

先看：

1. [离线推进与协议Mock方案.md](离线推进与协议Mock方案.md)
2. [调试与工作流.md](调试与工作流.md)
3. [配置管理.md](配置管理.md)

适用场景：

1. 只能做页面、配置、协议、mock
2. 想先补离线可闭环能力
3. 想知道哪些事该留到周末真机

### 1.4 周末上真机，想减少来回试错

先看：

1. [周末调试前核对单.md](周末调试前核对单.md)
2. [调试与工作流.md](调试与工作流.md)
3. [配置管理.md](配置管理.md)
4. [设备手册/README.md](设备手册/README.md)

适用场景：

1. 真机前先确认哪些能力已经接上
2. 要核对口位、坐标、时序、权限、驱动
3. 要看 K80 / M90 原厂说明、接线、结构图

### 1.5 要改 host、port、串口、模式或运行期配置

先看：

1. [配置管理.md](配置管理.md)
2. [调试与工作流.md](调试与工作流.md)

适用场景：

1. 改串口口位、波特率
2. 改 Socket host / port
3. 切 stub / real
4. 确认到底哪份配置正在生效

### 1.6 要看设备资料、接线图、原厂说明

先看：

1. [设备手册/README.md](设备手册/README.md)

适用场景：

1. 要看 K80 手册
2. 要看 M90 手册
3. 要找结构图、接口图、接线图、资源制作说明

### 1.7 要补提测依据、差异说明、历史记录

先看：

1. [需求变更记录.md](需求变更记录.md)
2. [旧项目功能执行文档.md](旧项目功能执行文档.md)
3. [新壳与旧4.4差异记录.md](新壳与旧4.4差异记录.md)

适用场景：

1. 要补这次需求改了什么
2. 要说明哪些旧行为继承，哪些不继承
3. 要给后面测试或下一次会话留证据

## 3. 当前建议的阅读顺序

如果是正常继续推进，建议固定按这个顺序看：

1. [README.md](README.md)
2. [新会话接手说明.md](新会话接手说明.md)
3. [推进步骤.md](推进步骤.md)
4. 再按当前任务跳到对应专项文档

这样不会再出现：

1. 不知道先看哪份
2. 同一件事来回翻多份文档
3. 把工作日任务和真机任务混在一起
