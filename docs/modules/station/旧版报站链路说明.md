# 旧版报站链路说明

这份文档只回答一件事：旧项目 IStationDevice 里的“报站”到底分成哪些链路，迁移时分别落在哪。

## 首页手动报站主链

旧入口文件：

- `app/src/main/java/com/lhxy/istationdevice/ui/activity/MainActivity.java`

首页按钮事件集中在：

- `onViewClicked()`

旧版手动报站直接依赖下面这些方法：

- `switchCurrentStation()`
- `repeatNewspaper()`
- `stopNewspaper()`
- `forwardNewspaper()`
- `backwardNewspaper()`
- `switchDirection()`

旧版首页报站的核心状态只有两份：

- `currentStation`：当前站点游标
- `isNextStation`：当前是在“下站预报”还是“本站到站”阶段

旧节奏是：

1. 初始态显示起点站，本站
2. 点一次报站，切到下一站预报
3. 再点一次报站，切到本站到站
4. 到终点站后自动切方向

## 竹节图和屏显链路

旧版首页视图和屏显发包是绑在一起的：

- `initBusView()`
- `controlBusLine(int busIndex)`
- `showBusLine(...)`
- `sendReportStationCommand(BusLineModel busLineModel)`
- `sendSwitchLine()`

旧版 485 串口收发链：

- `open485SerialPort()`
- `close485SerialPort()`
- `Send485Thread`

旧版协议生成器：

- `app/src/main/java/com/lhxy/istationdevice/serialport/ProtocolGenerate.java`
- `app/src/main/java/com/lhxy/istationdevice/serialport/TongDaProtocol.java`
- `app/src/main/java/com/lhxy/istationdevice/serialport/LHXYProtocol.java`
- `app/src/main/java/com/lhxy/istationdevice/serialport/HengWuProtocol.java`
- `app/src/main/java/com/lhxy/istationdevice/serialport/LEDProtocol.java`

旧版屏显分两类动作：

1. 切线时发线路名、线路状态、整线站序
2. 报站时发当前站报文，部分协议还会发内屏文案

## GPS 自动报站链

旧版 GPS 串口和自动报站主要在两个文件：

- `MainActivity.java`
- `mvp/main/MainPresenter.java`

页面侧入口：

- `openGPSPort()`
- `GPSThread`
- `gpsSerialPortResultListener`
- `automaticSwitchCurrentStation(GPSReportStationResult)`

判定核心在 `MainPresenter`：

- `getGpsRStationInfo(GPSOperationParam)`
- `duikaiRStationHandle(...)`
- `annularRStationHandle(...)`
- `antiReverseRStationHandle(...)`

状态输入模型：

- `GPSOperationParam`
- `GPSReportStationResult`
- `BusLineModel`
- `BusLineFriendRemindModel`

## 平台上报链

旧版页面里直接组站点相关上报：

- `assembleSiteInfoReport()`
- `assembleFirstStationReport()`
- `assembleSpeedReport()`
- `assembleLineSwitching()`

协议生成真正落在：

- `socket/request/Generate808ReqPackage.java`
- `socket/request/GenerateAL808ReqPackage.java`

## 语音 / TTS / 喇叭链

旧版报站音频基本都堆在 `MainActivity`：

文件播报链：

- `disposeVoice()`
- `addVoice()`
- `addExternalVoice()`
- `friendshipVoice()`
- `serviceVoice()`
- `addServiceVoice()`

TTS 链：

- `systemDisposeVoice()`
- `playReportStationTTS()`
- `playDispatchTTS()`

播放控制链：

- `initMediaPlayer()`
- `createNextMediaPlayer()`
- `stopBusVoicePlay()`
- `stopTTS()`

GPIO / 音量控制链：

- `controlHorn()`

## 旧站点资源字段

旧版站点 CSV 并不只是站名和经纬度。报站完整逻辑依赖这些字段：

- `站台编号`
- `报站语音`
- `站名`
- `经度`
- `纬度`
- `角度`
- `海拔`
- `进站广告`
- `出站广告`
- `进站提示`
- `出站提示`
- `进站扩展`
- `出站扩展`
- `限速`
- `站前里程`
- `是否大站`
- `外音开否`

提醒资源另有一套：

- `编号`
- `提醒语音`
- `经度`
- `纬度`
- `角度`
- `海拔`
- `站前里程`
- `外音开否`

## 新壳对应落点

本轮开始，新壳不再把这些行为塞回首页，而是按能力域拆分：

1. `modules/domain/.../domain/station`
   - 站态机
   - 旧屏显派发桥
   - 站点快照转换
2. `modules/domain/.../domain/gps`
   - GPS 监听
   - 自动报站判定
3. `modules/protocol/.../protocol/legacy`
   - 旧 485 屏显协议生成器
4. `app`
   - 首页状态展示
   - 报站中心动作入口
