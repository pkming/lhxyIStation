# 新版报站迁移步骤

这份文档说明 Android 11 新壳当前的报站执行步骤，以及后面继续补齐时的推荐顺序。

## 当前模块目录

本轮新建的报站模块目录：

- `modules/domain/src/main/java/com/lhxy/istationdevice/android11/domain/station`

当前文件职责：

- `LegacyStationSnapshotFactory`
  - 把运行期 `StationState` 转成旧屏显协议需要的 `BusLineSnapshot`
- `LegacyStationDisplayUseCase`
  - 根据 `basicSetup.serialSettings.rs485Protocol` 选择旧版屏显协议
  - 打开屏显串口
  - 发送线路名、整线站序、当前站报文
- `LegacyStationAudioUseCase`
   - 按旧资源目录拼接手动/自动报站音频序列
   - 在 MP3 与 TTS 之间按配置切换
   - 控制内音 / 外音 / 内喇叭 GPIO 与音量

## 新壳当前执行步骤

### 1. 资源准备

1. `StationResourceArchiveUseCase` 导入 `SourceFile.zip`
2. 资源解压到 app 管理目录
3. `LegacyGpsRouteCatalog` 从 `SourceFile/Bus` 读取线路和站点

### 2. 报站模块初始化

1. `ShellRuntime.applyConfig()` 同步配置
2. `TerminalModuleHub` 挂载 `StationBusinessModule`
3. `StationBusinessModule.onContextUpdated()`
   - 同步线路状态
   - 需要时刷新屏显整线数据

### 3. 手动报站

1. 页面触发 `station` 模块
2. `StationBusinessModule.runSample()` 或 `runAction("advance_station")`
3. `StationState.advanceStation()` 按旧版“下站预报 / 本站到站”节奏推进
4. `LegacyStationAudioUseCase.playManualStation()` 按当前阶段播放 MP3 或 TTS
5. `LegacyStationDisplayUseCase.sendCurrent()` 发送当前站屏显报文
6. `DvrSerialDispatchUseCase`
   - 发 GPS 上报
   - 发站点信息上报

### 4. 自动 GPS 报站

1. `GpsSerialMonitor` 产出实时定位快照
2. `LegacyGpsAutoReportEngine.evaluate()` 按旧线路属性做判定
3. `StationState.recordAutoStation()` 更新站态
4. `LegacyStationAudioUseCase`
   - 自动进站时按旧目录播放 MP3 或 TTS
   - 友情提醒优先走旧资源 MP3，缺文件时回退到 TTS
5. `LegacyStationDisplayUseCase.sendCurrent()` 刷新当前站屏显
6. `DvrSerialDispatchUseCase.sendSiteInfo()` 上报站点

### 5. 手动切方向

1. 页面触发 `runAction("switch_direction")`
2. `StationBusinessModule` 重新加载反向线路
3. `StationState.switchDirection(...)`
4. `LegacyStationDisplayUseCase.syncRoute(...)`
   - 重发线路名
   - 重发整线站序
   - 重发当前站

## 本轮已接通的能力

- 新建独立报站模块目录
- 旧站点资源字段扩展解析
- 手动报站按旧节奏推进
- 手动报站音频序列
- 手动切方向
- 重复报站
- 停止报站
- 自动 GPS 报站后刷新屏显
- 自动 GPS 报站音频 / 友情提醒音频
- MP3 / TTS 切换
- 内音 / 外音 / 内喇叭 GPIO 切换
- 报站音量 / TTS 音量控制
- 通达 / LHXY / 恒舞三套旧屏显协议的实时发送桥
- DVR GPS 上报
- DVR 站点信息上报

## 还要继续补的项

离“100% 还原”还差的重点在这里，后续继续按这个顺序推进：

1. 首页按钮节奏完全对齐旧终端
2. 服务音 / 发车提醒 / 超速语音
3. 真机核对 GPIO 电平与内外音实际效果
4. LED 导程牌和其他特殊 485 协议

## 推荐继续迁移顺序

1. 在 `domain/station` 补语音序列和 TTS 调度
2. 给首页“报站”按钮补旧版直达动作，不再只跳转模块页
3. 给报站中心补“回退一站 / 切方向 / 重复 / 停止 / 绑定 GPS”完整操作区
4. 真机联调 GPIO 喇叭和音量
5. 最后补服务音、友情提醒、LED 导程牌
