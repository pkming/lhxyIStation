## 4. 基础链路校验

这部分在自动报站恢复前也必须通过。

### 4.1 配置核对

必须确认：

1. `debugReplay.gpsSerialKey = gps`
2. `serial.gps.portName = ttyS5`
3. `serial.gps.baudRate = 115200`
4. `serial.gps.mode = real`

如果切过配置，要删除运行期缓存：

`/storage/emulated/0/Android/data/com.lhxy.istationdevice.android11/files/config/terminal-config.runtime.json`

### 4.2 日志核对

必须能看到：

1. `stty applied /dev/ttyS5 @115200`
2. `real open /dev/ttyS5 @115200`
3. `已绑定 GPS 串口监听: gps/ttyS5`
4. `gps raw sample` 中持续出现 `$GNRMC`、`$GNGGA`、`$GNGSA`

### 4.3 定位状态核对

通过标准：

1. `valid=true`
2. `fixQuality>=1`
3. `fixType=2(2D)` 或 `3(3D)`
4. `usedSatellites>0`
5. `latitude / longitude` 非空

未通过但不一定是代码问题的情况：

1. 有 NMEA，但一直 `valid=false`
2. `GSA` 一直是 `A,1`
3. `GGA` 的 `fixQuality=0`

这通常优先看天线、环境、冷启动等待时间。

### 4.4 页面核对

通过标准：

1. 首页 `GPS` 状态能切到“有效”
2. 首页卫星数和实时定位一致
3. 站点学习页“实时 GPS”经纬度同步刷新
4. 页面速度显示为 `km/h`
5. 海拔字段不出现 `M` 这种单位串误入值

## 5. 串口上报校验

这部分针对当前已恢复的 DVR 串口 GPS 上报。

### 5.1 前提

只有在下面条件同时满足时才校验：

1. `dispatchOwner = serial_rs232_1`
2. `rs2321Protocol = DVR`

### 5.2 通过标准

必须能看到：

1. 手动报站时出现 `DVR_GPS_REPORT`
2. 手动报站时出现 `DVR_SITE_INFO`
3. 周期上报日志持续出现 `DVR_GPS_REPORT`
4. 帧里经纬度不是空值，也不是 `0`

### 5.3 当前实现说明

当前项目已经恢复：

1. 手动报站触发 GPS 上报
2. 手动报站触发站点信息上报
3. 按 `network.infoInterval` 周期触发 DVR GPS 上报

## 6. 旧项目一致性校验

这部分才是“完整复刻”的标准，不是只有串口通了就算完成。

### 6.1 旧项目 GPS 真实消费链

旧项目里 GPS 不只是显示，它还参与：

1. 自动寻找最近站点
2. 进站 / 出站判定
3. 对开 / 环线 / 防反三种线路属性分支
4. 友情提醒点触发
5. 角度过滤
6. 站前里程阈值过滤
7. GPS 定时上报
8. GPS 校时
9. 站点学习保存回数据库

### 6.2 完整复刻通过标准

只有下面都成立，才能算 GPS 功能完整复刻：

1. 选用 `L1` 后，系统能根据实时定位自动找到最近站点
2. 能自动判定进站 / 出站
3. 能随着车辆移动自动切到下一站
4. 友情提醒点能按坐标和里程触发
5. DVR 周期 GPS 上报持续正常
6. 页面展示、串口上报、自动报站三者使用的是同一份实时 GPS 数据

### 6.3 当前还未通过的项

截至当前版本，下面这些还不能算通过：

1. 自动 GPS 报站状态机
2. 友情提醒点完整触发
3. GPS 校时
4. 站点学习真实落库
5. 平台网络 GPS 上报完整等价
